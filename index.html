      1. 用户存储关于账户密码的相关设置，一行对应一个账户的密码设置

      4. 有9个字段，每段也用“:”分搁

        1. 用户名：用户登录到系统时使用的名字，而且是惟一的。

        2. 口令：存放SHA512算法加密过的口令

          1. 若为空，
          2. 表示该用户不需口令即可登录；
          3. 若为！！，表示该用户从来没有设置过密码，不能登录系统；
          4. 若为*，表示该帐号被禁用；
          5. 若为！！MD5密码，则表示用户密码被passwd –l命令锁住；若为！MD5密码，则表示用户密码被usermod –L命令锁住。

        6. 口令在多少天后需要被修改，即口令多少天后到期。

        7. 口令到期前多少天内给用户发出警告。

    1. 用户管理命令

      1. 管理员管理用户帐号时主要完成三种基本工作，即合理、有效、安全地新建、删除和管理用户。

      2. useradd

        1. 修改用户的属性信息，只有超级用户root才能使用

        2. 格式：useradd [选项] <用户名>

        3. 选项：

          1. -u 用户ID 指定用户UID
          2. -g 组ID或组名 指定新用户的主组
          3. -G 组ID或组名 指定新用户的附加组
          4. -d 主目录 指定新用户的主目录
          5. -s 登录shell 指定新用户使用的shell，默认为/bin/bash
          6. -e 有效期限 指定用户的登录失效时间，例如：11/30/2012
          7. -f 缓冲天数：设置在密码过期后多少天关闭该帐号
          8. -c 备注 为账户加上备注
          9. -m 默认主目录 自动创建与用户名同名目录
          10. -n 取消建立以用户名称为名的组
          11. -r 建立系统帐号

        5. 注：由于新增加用户还未设置密码，因此不能使用该用户登陆

      4. passwd命令

        1. 格式： passwd [选项] <用户名>

        2. 功能：设置、修改用户的口令以及口令的属性

        3. 选项：

          1. -d （delete） 删除用户的口令，则该用户账号无需口令即可登录。
          2. -l （lock） 锁住口令。
          3. -u （unlock） 恢复禁用用户账号。
          4. -S （status） 显示指定用户账号的状态。
          5. -n days 设置口令多少天内不能修改。
          6. -x days 设置口令多少天后过期。
          7. -w days 设置口令过期前多少天发警告。
          8. -i days 设置口令过期后再宽限多少天。
          9. -e 让用户的口令立即过期。

        5. 注意：系统管理员可以设置所有用户的密码，普通用户只能修改自己的密码

      6. usermod命令

        1. 格式：usermod [选项] <用户名>

        2. 功能：修改用户的属性信息，只有超级用户才可以使用该命令。

        3. 选项：

          1. -g 组ID或组名 指定新用户的主组
          2. -G 组ID或组名 指定新用户的附加组
          3. -d 主目录 指定新用户的主目录
          4. -s 登录shell 指定新用户使用的shell，默认为bash
          5. -e 有效期限 指定用户的登录失效时间
          6. -u 用户ID 指定用户UID
          7. -c 全名 指定用户全称
          8. -f 缓冲天数 指定口令过期后多久将关闭此账号
          9. -l 用户名 指定用户的新名称
          10. -L 用户名 锁定用户密码，使密码无效
          11. -U 用户名 解除密码锁定。

        5. 注意：usermod命令与useradd命令的区别在于 usermod命令可以修改用户名且在禁用和恢复账号功能上，命令usermod不等同于 passwd。

      8. userdel命令

        1. 格式：userdel [选项] <用户名>

        2. 功能：删除指定的用户账号，只有超级用户才能使用该命令。

        3. 选项：

          1. -r 用于删除用户的Home目录和邮件
          2. -f 强制删除用户登录目录及目录中的所有文件

        5. 注意：正在使用系统的用户不能被删除，必须先终 止该用户的所有进程才能删除该用户。

      10. 成批添加用户：newusers

        1. 第一步:先建立文本文件：n_user.txt

          1. ln:student:1101:100:lining:/home/ln:/bin/bash
          2. cm:student:1102:100:chenmeng:/home/cm:/bin/bash
          3. nly:student:1103:100:niuliyue:/home/nly:/bin/bash
          4. zzz:student:1104:100:zhangzhenzhen:/home/zzz:/bin/bash

        3. 第二步:执行命令newusers n_user.txt

        4. 注意：使用这种方式的缺点是文件中用户的密码是明文，都为student。

      12. 成批修改口令：chpasswd（该命令在CentOS 7.3下不生效，在CentOS 8.1下生效）

        1. 第一步:先建立文本文件：n_uspw.txt

          1. ln:123456
          2. cm:123456
          3. nly:123456
          4. zzz:123456

        3. 第二步:执行命令chpasswd<n_uspw.txt

      14. 其他的shell命令

        1. id命令

          1. 格式：id [选项] [用户名]

          2. 功能：查看一个用户的UID和GID

          3. 选项：

            1. -g 显示用户的主组的GID
            2. -G 显示用户所有组的GID
            3. -u 只显示UID

        3. 2 ）whoami命令

          1. 格式：whoami
          2. 功能：用于显示当前用户的名称。

        5. 3）su命令

          1. 格式：su [-] [用户名]

          2. 功能：转换当前用户到指定的用户账号。

          3. 选项：

            1. - 在转换当前用户的同时转换用户工作环境；
            2. 用户名 若不指定用户名则转换当前用户到 root。
